# 快速启动指南

## 5 分钟快速部署

### 1. 克隆项目

```bash
git clone https://github.com/your-username/tiktok-monitor.git
cd tiktok-monitor
```

### 2. 一键部署

```bash
chmod +x deploy.sh
./deploy.sh
```

### 3. 访问应用

打开浏览器访问：http://localhost

## 首次使用

### 1. 配置系统设置

点击左侧菜单「系统设置」，配置：

- **默认监控间隔**：240 分钟（4小时）
- **最大并发检查数**：5
- **默认监控视频数**：20

### 2. 添加代理（可选）

点击「代理管理」→「添加代理」：

```
格式：ip:port:username:password
示例：
192.168.1.100:1080
192.168.1.101:1080:user:pass
```

支持批量添加（每行一个）。

### 3. 创建项目

点击「项目管理」→「添加项目」：

```
项目名称：竞品监控
描述：监控竞品账号数据
```

### 4. 添加账号

点击「账号列表」→「添加账号」：

- **所属项目**：选择刚创建的项目
- **用户名**：TikTok 账号用户名（必填）
- **监控间隔**：使用默认值或自定义
- **代理设置**：选择代理或留空随机
- **视频监控**：默认开启

### 5. 立即检查

添加账号后会自动触发首次检查，也可以手动点击「立即检查」。

### 6. 查看数据

- **账号列表**：查看所有账号的当前数据和趋势图
- **账号详情**：点击账号查看详细历史数据和视频列表

## 常用操作

### 批量添加账号

1. 准备 CSV 文件，格式：

```csv
username,nickname,monitor_interval
user1,昵称1,240
user2,昵称2,240
```

2. 点击「导入」→ 选择文件 → 上传

### 批量操作

1. 勾选多个账号
2. 点击批量按钮：
   - 批量启用
   - 批量禁用
   - 批量检查
   - 批量删除

### 查看趋势

点击账号进入详情页，可以查看：

- 粉丝数趋势
- 关注数趋势
- 点赞数趋势
- 视频数趋势

## 故障排查

### 服务启动失败

```bash
# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart
```

### 账号检查失败

常见原因：
- 用户名错误
- 代理失效
- API 限流

解决方法：
- 确认用户名正确
- 更换代理或使用本地 IP
- 降低并发数或增加监控间隔

### 数据不更新

1. 检查账号是否启用
2. 查看最后检查时间
3. 手动触发「立即检查」
